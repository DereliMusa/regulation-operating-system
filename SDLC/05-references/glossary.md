# Glossary

Domain and project terms used across the codebase and docs.

| Term | Meaning |
|---|---|
| MDR | Regulation (EU) 2017/745 on medical devices |
| IVDR | Regulation (EU) 2017/746 on in-vitro diagnostic devices |
| GSPR | General Safety and Performance Requirements (MDR Annex I) |
| Technical file | Complete documentation set demonstrating device conformity |
| Device class | Risk class: MDR I/IIa/IIb/III, IVDR A/B/C/D |
| Notified Body (NB) | Independent body performing conformity assessment |
| CER | Clinical Evaluation Report |
| PMS | Post-Market Surveillance |
| PMCF | Post-Market Clinical Follow-up |
| PSUR | Periodic Safety Update Report |
| Risk management | ISO 14971 process: identify, evaluate, mitigate, verify hazards |
| Traceability | Links between requirement, risk, test, and clinical evidence |
| Change impact | Analysis of what a change affects across linked artifacts |
| CAPA | Corrective and Preventive Action |
| UDI | Unique Device Identification |
| EUDAMED | European database on medical devices |
| Auditor Simulation | Certra feature that reviews a file like an auditor to surface deficiencies |
| Readiness | Percentage indicating how submission-ready a technical file is |
| Conformity | GSPR status: conforming / partial / missing |
| RA | Regulatory Affairs (persona/role) |
| QA | Quality Assurance (persona/role) |
| SDLC | This knowledge base and the session protocol around it |
| Gate A / Gate B | Documentation phase / implementation phase |
| MVP / Phase 1 / Phase 2 | Fast demo / full product / sellable SaaS |
